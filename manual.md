## 시작하기

**납품 관리 Pro**는 소규모 납품 사업자를 위한 오프라인 우선 PWA 앱입니다.  
인터넷 없이도 사용 가능하며, Firebase를 연결하면 여러 기기에서 실시간 동기화됩니다.

### 첫 실행

1. 브라우저에서 앱 주소를 열면 바로 사용 가능합니다.
2. 데이터는 기기 내 `localStorage`에 자동 저장됩니다.
3. 여러 기기에서 함께 사용하려면 **[Firebase 동기화](#firebase-동기화)** 설정이 필요합니다.

## 화면 구성

앱 하단 탭바로 각 기능을 이동합니다.

| 탭 | 기능 |
|---|---|

### 상단 버튼

- **🌙 다크모드**: 밝기 전환
- **🔗 Firebase**: 워크스페이스 연결·해제

## 거래처 관리

### 거래처 추가

1. **통계** 탭 또는 **거래처** 탭에서 `+ 거래처 추가` 버튼 클릭
2. 이름, 연락처, 메모 입력 후 저장

### 공유 거래처 자동 추가 *(v95 신규)*

공유 워크스페이스를 등록하면 상대방이 공개한 거래처가 납품 입력 시 **자동으로 내 거래처 목록에 추가**됩니다.

- 거래처 카드에 `📦 워크스페이스명` 뱃지가 표시됩니다
- 직접 등록하지 않아도 납품 입력·조회가 즉시 가능합니다
- 공유 해제 시 납품 내역이 없는 거래처는 자동 삭제됩니다

### 거래처 수정·삭제

- 거래처 카드를 탭하면 상세 화면으로 이동
- 상세 화면에서 ✏️ 수정 / 🗑️ 삭제 가능
- 납품 이력이 있는 거래처는 **삭제 대신 숨기기** 권장

## 납품 입력

### 새 납품 등록

1. 하단 중앙 🚛 버튼 클릭
2. **거래처 선택** — 검색창에 입력하면 두 가지 섹션이 표시됩니다:
   - 내 거래처 목록
   - `📦 공유 거래처` — 상대방이 공개한 거래처 (직접 등록 불필요)
3. **날짜** 설정 (기본값: 오늘)
4. **품목 추가**: 품목명 / 수량 / 단가 입력
   - 이전에 납품한 품목명은 자동완성으로 빠르게 입력 가능
   - 단가는 거래처별 최근 단가가 자동 적용
5. **저장** 버튼 클릭

> 💡 공유 거래처를 처음 선택하면 내 거래처 목록에 자동 추가됩니다.  
> 이후부터는 일반 거래처처럼 바로 선택 가능합니다.

### Enter 키 빠른 입력

납품 입력 시 키보드 **Enter(또는 이동) 키**로 다음 필드로 자동 이동하며, 화면도 해당 필드로 함께 스크롤됩니다.

| 순서 | 현재 필드 | Enter 동작 |
|---|---|---|
| 1 | 거래처 | → 날짜로 이동 |
| 2 | 날짜 | → 품목명으로 이동 |
| 3 | 품목명 | → 수량으로 이동 |
| 4 | 수량 | → 단가로 이동 |
| 5 | 단가 | → 품목 추가 실행 |

> 💡 품목을 여러 개 연속으로 입력할 때 화면을 직접 스크롤하지 않아도 입력창이 자동으로 보입니다.

### 반품 / 회수 처리 *(v105 신규)*

거래처에 납품한 물품이 반품되거나 회수되었을 때 사용합니다.

1. 하단 중앙 🚛 버튼 클릭
2. 거래처 선택 및 품목 입력 (반품할 품목·수량 동일하게 입력)
3. 확정 팝업에서 **↩ 반품/회수** 토글 ON
4. **확정** 버튼 클릭

> ⚠️ 반품/회수 토글과 타인거래 토글은 동시에 켤 수 없습니다.

**반품 처리 결과**

| 항목 | 동작 |
|---|---|
| 재고 | 반품 수량만큼 자동 **입고(복원)** |
| 미수금 | 해당 거래처 미수금에서 자동 **차감(상쇄)** |
| 정산·명세서 | 별도 작업 없이 합계에 자동 반영 |
| 수금 버튼 | 반품 전표에는 비활성화 (차감만 발생) |

반품 전표는 내역·상세·거래명세서에 **↩ 반품** 배지로 표시됩니다.

### 납품 수정·삭제

- 내역 탭 또는 거래처 상세에서 전표를 탭
- **수정**: 품목·수량·날짜 변경 후 저장
- **삭제**: 삭제 버튼 → 이중 확인 후 삭제

### 수금 처리

전표 상세 화면 하단에서 수금 방식 선택:

| 방식 | 설명 |
|---|---|
| 현금 | 현금 수령 |
| 이체 | 계좌이체 |
| 혼합 | 현금 + 이체 분리 입력 |
| 할인 완납 | 할인 금액 차감 후 완납 처리 |

- 부분 수금: 수금액을 납품 총액보다 적게 입력하면 미수금으로 남음
- 수금 취소: 수금 완료된 전표에서 `수금 취소` 버튼 클릭

> 💡 **CRM 연동 시**: 거래장에서 수금 처리하면 CRM에도 즉시 반영됩니다. *(v95)*

### 메모 입력

납품 전표 카드에서 **📝 메모** 버튼을 탭하면 해당 전표에 메모를 추가할 수 있습니다.

- 메모가 있는 전표는 주황색 테두리로 표시
- 메모는 거래처 명세표에서도 함께 표시됩니다

## 미수금 관리

**더보기 → 미수금** 탭에서 전체 미수금 현황을 확인합니다.

### 경과일 필터

| 필터 | 대상 |

## 납품 내역

**내역** 탭에서 전체 납품 기록을 조회합니다.

### v96 변경사항 — 공유 거래처 직접 납품

B가 공유 거래처에 납품을 입력하면 **A의 Firebase에 직접 저장**되고, A 화면에 실시간으로 반영됩니다.

| 항목 | 내용 |
|---|---|
| A 화면 실시간 반영 | B 납품 저장 즉시 A 내역 탭에 표시 (📦 토스트 알림) |
| B 수정·삭제 권한 | B가 직접 입력한 공유 납품은 B도 수정·삭제 가능 |
| 합산 표시 | A·B 납품 내역이 내역 탭에서 함께 합산 표시 |

### v95 변경사항 — 공유 내역 합산 표시

공유 워크스페이스를 등록한 경우, 내역 탭에서 **내 납품 내역 + 상대방 공유 납품 내역**이 함께 표시됩니다.

| 표시 | 의미 |
|---|---|
| 배지 없음 | 내가 입력한 납품 (수정·결제 가능) |
| `📦 워크스페이스명` 배지 | 공유된 상대방 납품 (읽기 전용) |

- 공유 내역은 수정·삭제·결제 버튼이 표시되지 않습니다
- 상대방 전표 오류는 해당 담당자가 직접 수정해야 합니다

### 타인거래 처리

납품을 내 매출이 아닌 **타인(다른 담당자) 명의의 거래**로 처리할 수 있습니다.

- 납품 확정 팝업에서 **타인거래** 토글 ON → 전표에 👤 배지 표시
- 타인거래 전표는 **재고 차감 없음**, 내 매출 집계에서 제외
- 내역 탭에서 `↩ 내거래로` 버튼으로 언제든 복원 가능 (복원 시 재고 수동 확인 필요)
- 반품/회수 토글과 동시에 사용 불가

### v102 변경사항 — 공유 대납 표시 로직 개선

공유 거래처에 B가 대신 납품 시, 이전에는 같은 거래처를 구독 중인 제3자(C)에게도 노출되던 버그를 수정했습니다.

- 대납 전표는 **납품한 본인(B) + 거래처 담당자(A) 앱에만** 표시
- 대납을 한 B의 매출 집계에도 정상 포함 (이전에는 제외되던 버그 수정)

### 검색·필터

- 상단 검색창: 거래처명 / 품목명 검색
- 기간 필터: 월별 선택 또는 직접 기간 지정
- 미수금 전표만 보기: 필터에서 `미수` 선택

### 정렬

- 날짜 최신순 (기본)
- 날짜 오래된순
- 금액 높은순

## 재고 관리

**더보기 → 재고** 탭에서 품목별 재고를 관리합니다.

### 품목 추가

1. `+ 품목 추가` 버튼 클릭
2. 품목명, 단위(박스/kg/개 등), 초기 수량, 안전 재고량 입력

### 재고 조정

- **입고**: 수량 직접 입력 또는 `+` 버튼
- **출고**: 납품 저장 시 연결된 품목의 재고가 자동 차감
- **수동 조정**: 품목 탭 → `조정` 버튼 → 증감량 입력 + 사유 메모

### 재고 재계산 (🔄 버튼)

재고 탭 상단의 🔄 버튼을 탭하면 **오늘 납품 전표를 기준으로 재고를 재계산**합니다.

- 오늘 전표만 대상으로 처리 (과거 전표는 포함하지 않음)
- 반복 탭 시 중복 차감 방지

### 재고 과거 조회

재고 탭 상단 ◀▶ 버튼 또는 날짜 선택으로 과거 재고를 조회할 수 있습니다.

> ⚠️ 재고 이력은 **어제·오늘**만 보관됩니다. 그 이전 날짜는 조회가 제한됩니다.

### 재고 이력

- 품목 탭에서 `이력 보기` 클릭
- 일자별 입고·출고·조정 내역 확인 가능
- 이력 보존 기간: Firebase 연결 시 30일 / 오프라인 최근 2일

> 💡 **다음날 이월은 그날 "실제로 가장 마지막에 일어난 변동"을 기준으로 계산됩니다.** 납품등록·납품수정·입고·출고가 어떤 순서로 일어났든, 같은 날 안에서 실제로 가장 늦게 일어난 변동의 결과값이 다음날 아침 이월 재고가 됩니다. *(v107부터 정상 동작 — 이전 버전에서는 납품수정이 납품등록보다 늦게 일어나도 이월 계산에서 더 이른 시점 값이 잘못 채택되는 버그가 있었습니다.)*

### 안전 재고 알림

- 안전 재고량 설정 시 재고가 그 이하로 떨어지면 **🟠 노란색** 표시
- 위험 재고량 이하 시 **🔴 빨간색** 표시

## 정산서

**더보기 → 정산** 탭에서 거래처별 월별 정산서를 생성합니다.

1. 거래처 선택 → 정산 월 선택
2. 해당 월의 납품 내역이 자동 집계
3. **PNG 저장**: 정산서를 이미지로 저장 (카카오톡 전송 등)

## 자동 백업

**더보기 → 백업** 탭에서 자동 백업을 설정합니다.

### 백업 주기

- 매일 / 매주 / 매월 선택 가능
- 설정된 주기마다 자동으로 JSON 백업 파일 생성

### 백업 저장 위치

- **기기 다운로드 폴더**: 기본값
- **폴더 지정**: `저장 위치 변경` 버튼으로 특정 폴더 선택 (File System Access API 지원 브라우저)

### 수동 백업·복원

- **설정 → 데이터 관리 → JSON 백업 내보내기**: 즉시 백업
- **설정 → 데이터 관리 → JSON 가져오기**: 백업 파일로 복원

## Firebase 동기화

여러 기기(스마트폰·태블릿·PC)에서 데이터를 공유할 수 있습니다.

### 워크스페이스 연결

1. 상단 우측 🔗 버튼 클릭
2. **워크스페이스 ID** 입력 (영문 소문자·숫자 조합, 본인만 아는 값)
3. `🔌 연결` 버튼 클릭
4. 처음 연결 시: 로컬 데이터를 서버에 올릴지, 서버 데이터를 내려받을지 선택

> ⚠️ 같은 워크스페이스 ID를 입력한 기기끼리 데이터를 공유합니다.  
> 타인이 추측하기 어려운 ID를 사용하세요.

### 연결 상태 표시

| 상태 | 의미 |
|---|---|
| 🟢 온라인 동기화 | 정상 연결됨, 실시간 동기화 중 |
| 🟡 동기화 중 | 데이터 업로드/다운로드 중 |
| 🔴 동기화 오류 | 연결 끊김, 자동 재연결 시도 중 |

## 거래처 공유

> **언제 사용하나요?**  
> 동일한 거래처(예: Q마트)를 A·B 두 담당자가 각자 납품할 때,  
> A가 Q마트를 공유 설정하면 B는 **거래처를 직접 등록하지 않고도** Q마트에 납품 등록·조회가 가능합니다.

---

### 개념 이해

```
A의 납품앱                    B의 납품앱
  Q마트 → 공유 설정 ✅          A의 워크스페이스 ID 등록
       ↓                              ↓
  B가 납품 등록 시 거래처 검색
  → "📦 공유 거래처" 섹션에 Q마트 표시
  → 선택 시 B의 거래처 목록에 자동 추가
  → 이후 납품 등록·조회 모두 가능
```

---

### A — 거래처 공유 설정 (공개하는 쪽)

1. 상단 🔗 버튼 클릭
2. **\"공유 거래처 설정\"** 섹션 확인
3. 공개할 거래처 옆 체크박스 활성화
4. Firebase에 즉시 반영됩니다

> ⚠️ 공개 설정한 거래처만 상대방에게 보입니다.  
> 공개 해제하면 상대방 앱에서 즉시 사라집니다.

---

### B — 공유 워크스페이스 등록 (공유받는 쪽)

1. 상단 🔗 버튼 클릭
2. **\"거래처 내역 공유\"** 섹션 확인
3. **이름** (예: 박우석) 과 **워크스페이스 ID** 입력
4. **➕ 추가** 버튼 탭
5. 이후 납품 입력 시 거래처 검색하면 `📦 공유 거래처` 섹션에 표시됨

---

### B가 공유 거래처로 할 수 있는 것

| 기능 | 가능 여부 |
|---|---|
| 납품 입력 | ✅ 가능 (거래처 직접 등록 불필요) |
| A의 이전 납품 내역 조회 | ✅ 가능 (읽기 전용) |
| 내역 탭에서 합산 확인 | ✅ 가능 |

## 자주 묻는 질문

**Q. 공유 거래처가 납품 검색에 안 나와요**  
A. ① Firebase에 연결된 상태인지 확인 (🟢 온라인) ② 상대방이 거래처 공개 설정을 했는지 확인 ③ 워크스페이스 ID가 정확한지 확인. 앱을 닫고 다시 열면 갱신됩니다.

**Q. 공유 워크스페이스를 등록했는데 합산이 안 돼요**  
A. ① 상대방의 워크스페이스 ID가 정확한지 ② Firebase에 연결된 상태인지 ③ 거래처명이 정확히 동일한지 (띄어쓰기·대소문자 포함) 확인 후 명세표를 닫고 다시 열어보세요.

**Q. 상대방이 입력한 내역에 결제 처리를 하고 싶어요**  
A. 공유 내역은 읽기 전용입니다. 결제 처리는 해당 전표를 입력한 담당자가 직접 처리해야 합니다.

**Q. 공유했더니 내 다른 거래처 내역도 보이나요?**  
A. 아닙니다. 공개 설정한 거래처의 내역만 상대방에게 보입니다. 다른 거래처 데이터는 노출되지 않습니다.

**Q. CRM에서 수금 처리했는데 거래장에 반영이 안 돼요**  
A. 거래장과 CRM이 모두 Firebase에 연결된 상태인지 확인하세요. 연결 상태에서 CRM 수금 처리 시 거래장 전표가 자동으로 업데이트됩니다.

**Q. 데이터가 사라졌어요**  
A. Firebase에 연결되어 있다면 🔗 버튼 → 다시 연결하면 서버에서 복원됩니다. Firebase 미연결 상태에서 브라우저 데이터를 삭제했다면 마지막 JSON 백업 파일로 복원하세요.

**Q. 두 기기에서 동시에 입력하면 어떻게 되나요?**  
A. 마지막으로 저장한 기기의 데이터가 우선 적용됩니다. 충돌 방지를 위해 동시 입력은 피해주세요.

**Q. 오프라인에서 입력한 내용이 온라인 복귀 후 사라져요**  
A. 설정 탭 → 워크스페이스 잠금(🔒)을 활성화하면 온라인 복귀 시 오프라인 변경사항을 자동으로 서버에 업로드합니다.

**Q. 백그라운드에 내려놨다가 돌아오면 동기화가 멈추거나 "동기화 중"에서 안 풀려요**  
A. 모바일 OS가 백그라운드 상태에서 Firebase 연결을 조용히 끊어버리는 경우, 앱은 이를 바로 알아채지 못해 동기화가 무한 대기 상태에 빠질 수 있었습니다. 이 문제를 해결했습니다 (자세한 내용은 변경이력 v106 참고). 그래도 한 번씩 멈춘다면 설정 → Firebase에서 재연결 버튼을 눌러주세요. 반복된다면 **🩺 동기화 진단 로그**(상단 🔗 버튼 안)를 복사해서 문의해주시면 원인 파악에 도움이 됩니다.

**Q. 납품 입력 시 Enter를 눌러도 다음 칸으로 안 넘어가요**  
A. 앱을 최신 버전(v105 이상)으로 업데이트하세요. 거래처 검색창과 품목명·수량·단가 입력란 모두에서 화면(HTML)에 직접 박혀있던 Enter 처리와 JS의 공통 처리가 동시에 등록되어 간헐적으로 포커스 이동이 씹히던 버그가 있었으며, 중복 처리를 제거하고 JS 쪽 공통 처리로 통일해 수정되었습니다.

**Q. Enter로 다음 필드로 이동은 되는데 화면이 안 따라와요**  
A. 동일하게 v105에서 수정된 내용입니다. Enter 이동 시 해당 입력창이 화면 중앙으로 자동 스크롤됩니다.

**Q. 반품 처리 후 재고가 안 늘어났어요**  
A. 반품/회수 토글을 ON으로 하고 확정했는지 확인하세요. 토글이 꺼진 상태로 확정하면 일반 납품으로 처리됩니다. 이미 잘못 등록했다면 해당 전표를 삭제하고 다시 등록하세요.

**Q. 타인거래로 등록했는데 내 매출에 포함됐으면 해요**  
A. 내역 탭에서 해당 전표 → `↩ 내거래로` 버튼을 탭하면 내 매출에 포함됩니다. 단, 재고는 자동 복원되지 않으므로 재고탭에서 수동으로 확인·조정하세요.

**Q. 앱이 느려졌어요**  
A. 설정 → 저장공간 관리 → 재고 이력 즉시 정리 또는 오래된 전표 정리를 실행해 보세요.

## 변경이력

### v119 (2026-06)

#### 동기화 안정성

- **중요 저장 즉시 업로드(선제 동기화)** — 납품 등록·거래처 추가 등 중요한 저장 시 debounce(800ms) 대기 없이 즉시 Firebase 업로드를 시작합니다. 사용자가 앱을 백그라운드로 전환하기 전에 이미 업로드가 완료된 상태를 만들어 `flushSync` 타임아웃 빈발 문제를 근본적으로 줄입니다.
- **`_saveAndFlush` 단순화** — `saveData(true)`가 즉시 업로드를 포함하므로 중복 `_flushSync` 호출 제거.

### v118 (2026-06)

#### 동기화 안정성

- **`_patchCrmWithRetry` 반환값 분기 수정** — `null`(CRM 미연동)이 falsy라 `if(ok)`에서 오류와 같이 처리되던 버그 수정. `true`/`null`/`false` 명시 분기로 변경해 미연동 주문에 대한 불필요한 재시도 제거, `false` 반환 시에도 재시도 큐에 정상 등록.
- **`orders.once()` 타임아웃 시 `_syncGuard` 즉시 해제** — 서버 조회(delta/full) 타임아웃 발생 시 `_syncGuard`가 최대 16초간 잠기던 문제 수정. 타임아웃 즉시 `_syncGuard = false`로 해제하고 동기화를 중단해 다음 시도가 즉시 가능하도록 개선.

### v117 (2026-06)

#### 동기화 안정성

- **flushSync 조기 중단** — 백그라운드 진입 직후 소켓이 끊기면 5초 타임아웃까지 기다리지 않고 즉시 hash 롤백 후 종료합니다. 포그라운드 복귀 시 자동 재시도됩니다.
- **소켓 안정화 대기 시간 연장** — `_refreshSocket` 후 write 시도 전 대기 시간을 600ms → 2000ms로 늘려 소켓이 완전히 안정되기 전에 업로드가 시작되어 타임아웃되던 문제를 해소합니다.
- **write 직전 연결 상태 재확인** — 동기화 직전 `isConnected`가 false면 즉시 hash 롤백하고 8초 타임아웃 없이 종료합니다.

### v114 (2026-06)

#### 공유 납품 오프라인 큐 개선

- **dead 큐(업로드 실패 보관함) 신설** — 자동 재시도(최대 5회)에도 계속 실패한 공유 납품을 폐기하지 않고 별도 보관합니다. 헤더 배지(`⚠️ 업로드 실패 N건`)를 탭하면 실패 목록을 확인하고 재시도하거나 삭제할 수 있습니다.
- **큐 중복 방지** — 오프라인 상태에서 같은 전표를 여러 번 저장하면 `order.id` 기준으로 덮어써 중복 업로드를 방지합니다.
- **재시도 백오프** — 자동 재시도는 5초 → 15초 → 1분 → 5분 → 10분 간격으로 늘어납니다. 5회 초과 시 dead 큐로 이동됩니다.

#### CRM 결제 동기화 내구성 강화

- **CRM 실패 큐 신설** — 지수 백오프 재시도(최대 5회)에도 CRM 패치에 실패하면 `_crmPatchFailQueue`에 저장합니다. 앱 재시작 또는 Firebase 재연결 시 자동으로 재시도해 결제 누락을 방지합니다.
- **반환값 구분 명확화** — `_patchCrmTransaction`이 `true`(성공) / `null`(CRM 미연동 주문, 재시도 불필요) / `false`(오류, 재시도 필요) 세 값으로 반환해 불필요한 재시도를 줄입니다.
- **Firebase 재연결 시 CRM 큐 자동 처리** — 온라인 복귀 시 공유 납품 큐 플러시와 함께 CRM 실패 큐도 자동으로 재시도됩니다.

#### UI 안정성

- **토스트 큐 시스템 도입** — 동시에 여러 토스트 메시지가 발생해도 겹치지 않고 순서대로 표시됩니다.
- **커스텀 확인 모달(customConfirmModal) 추가** — `confirm()` 대신 앱 스타일에 맞는 확인/취소 모달을 사용합니다. dead 큐 삭제 등 중요 작업 전 확인에 활용됩니다.
