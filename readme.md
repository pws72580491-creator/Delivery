## 시작하기

**납품 관리 Pro**는 소규모 납품 사업자를 위한 오프라인 우선 PWA 앱입니다.  
인터넷 없이도 사용 가능하며, Firebase를 연결하면 여러 기기에서 실시간 동기화됩니다.

### 첫 실행

1. 브라우저에서 앱 주소를 열면 바로 사용 가능합니다.
2. 데이터는 기기 내 `localStorage`에 자동 저장됩니다.
3. 여러 기기에서 함께 사용하려면 **[Firebase 동기화](#firebase-동기화)** 설정이 필요합니다.

## 화면 구성

앱 하단 탭바로 각 기능을 이동합니다.

| 탭 | 기능 |
|---|---|
| 📊 통계 | 이달 매출·미수금 요약, 최근 7일 바차트, 최근 납품 목록 |
| 🏪 거래처 | 거래처 목록 관리 |
| 🚛 (중앙 버튼) | 새 납품 입력 |
| 📋 내역 | 전체 납품 이력 검색·조회 |
| ··· 더보기 | 미수금 / 재고 / 정산 / 백업 / 설정 |

### 상단 버튼

- **🌙 다크모드**: 밝기 전환
- **🔗 Firebase**: 워크스페이스 연결·해제

## 거래처 관리

### 거래처 추가

1. **통계** 탭 또는 **거래처** 탭에서 `+ 거래처 추가` 버튼 클릭
2. 이름, 연락처, 메모 입력 후 저장

### 거래처 수정·삭제

- 거래처 카드를 탭하면 상세 화면으로 이동
- 상세 화면에서 ✏️ 수정 / 🗑️ 삭제 가능
- 납품 이력이 있는 거래처는 **삭제 대신 숨기기** 권장

### 거래처 숨기기

- 거래처 카드 우측 `숨기기` 버튼으로 목록에서 숨김 처리
- 숨겨진 거래처의 납품 이력은 내역에서 계속 조회 가능
- 숨김 해제: `표시` 버튼 클릭

## 납품 입력

### 새 납품 등록

1. 하단 중앙 🚛 버튼 클릭
2. **거래처 선택** (기존 거래처 목록 또는 직접 입력)
3. **날짜** 설정 (기본값: 오늘)
4. **품목 추가**: 품목명 / 수량 / 단가 입력
   - 이전에 납품한 품목명은 자동완성으로 빠르게 입력 가능
   - 단가는 거래처별 최근 단가가 자동 적용
5. **저장** 버튼 클릭

### 납품 수정·삭제

- 내역 탭 또는 거래처 상세에서 전표를 탭
- **수정**: 품목·수량·날짜 변경 후 저장
- **삭제**: 삭제 버튼 → 이중 확인 후 삭제

### 수금 처리

전표 상세 화면 하단에서 수금 방식 선택:

| 방식 | 설명 |
|---|---|
| 현금 | 현금 수령 |
| 이체 | 계좌이체 |
| 혼합 | 현금 + 이체 분리 입력 |
| 할인 완납 | 할인 금액 차감 후 완납 처리 |

- 부분 수금: 수금액을 납품 총액보다 적게 입력하면 미수금으로 남음
- 수금 취소: 수금 완료된 전표에서 `수금 취소` 버튼 클릭

## 미수금 관리

**더보기 → 미수금** 탭에서 전체 미수금 현황을 확인합니다.

### 경과일 필터

| 필터 | 대상 |
|---|---|
| 전체 | 모든 미수 전표 |
| 30일 이상 | 🟠 주의 |
| 60일 이상 | 🔴 위험 |
| 90일 이상 | 🚨 긴급 |

- 거래처를 탭하면 해당 거래처의 미수 전표 목록으로 이동
- 연락처가 등록된 거래처는 전화 아이콘으로 바로 전화 가능

## 납품 내역

**내역** 탭에서 전체 납품 기록을 조회합니다.

### 검색·필터

- 상단 검색창: 거래처명 / 품목명 검색
- 기간 필터: 월별 선택 또는 직접 기간 지정
- 미수금 전표만 보기: 필터에서 `미수` 선택

### 정렬

- 날짜 최신순 (기본)
- 날짜 오래된순
- 금액 높은순

## 재고 관리

**더보기 → 재고** 탭에서 품목별 재고를 관리합니다.

### 품목 추가

1. `+ 품목 추가` 버튼 클릭
2. 품목명, 단위(박스/kg/개 등), 초기 수량, 안전 재고량 입력

### 재고 조정

- **입고**: 수량 직접 입력 또는 `+` 버튼
- **출고**: 납품 저장 시 연결된 품목의 재고가 자동 차감
- **수동 조정**: 품목 탭 → `조정` 버튼 → 증감량 입력 + 사유 메모

### 재고 이력

- 품목 탭에서 `이력 보기` 클릭
- 일자별 입고·출고·조정 내역 확인 가능
- 이력 보존 기간: Firebase 연결 시 30일 / 오프라인 최근 2일

### 안전 재고 알림

- 안전 재고량 설정 시 재고가 그 이하로 떨어지면 **🟠 노란색** 표시
- 위험 재고량 이하 시 **🔴 빨간색** 표시

## 정산서

**더보기 → 정산** 탭에서 거래처별 월별 정산서를 생성합니다.

1. 거래처 선택 → 정산 월 선택
2. 해당 월의 납품 내역이 자동 집계
3. **PNG 저장**: 정산서를 이미지로 저장 (카카오톡 전송 등)

## 자동 백업

**더보기 → 백업** 탭에서 자동 백업을 설정합니다.

### 백업 주기

- 매일 / 매주 / 매월 선택 가능
- 설정된 주기마다 자동으로 JSON 백업 파일 생성

### 백업 저장 위치

- **기기 다운로드 폴더**: 기본값
- **폴더 지정**: `저장 위치 변경` 버튼으로 특정 폴더 선택 (File System Access API 지원 브라우저)

### 수동 백업·복원

- **설정 → 데이터 관리 → JSON 백업 내보내기**: 즉시 백업
- **설정 → 데이터 관리 → JSON 가져오기**: 백업 파일로 복원

## Firebase 동기화

여러 기기(스마트폰·태블릿·PC)에서 데이터를 공유할 수 있습니다.

### 워크스페이스 연결

1. 상단 우측 🔗 버튼 클릭
2. **워크스페이스 ID** 입력 (영문 소문자·숫자 조합, 본인만 아는 값)
3. `🔌 연결` 버튼 클릭
4. 처음 연결 시: 로컬 데이터를 서버에 올릴지, 서버 데이터를 내려받을지 선택

> ⚠️ 같은 워크스페이스 ID를 입력한 기기끼리 데이터를 공유합니다.  
> 타인이 추측하기 어려운 ID를 사용하세요.

### 연결 상태 표시

| 상태 | 의미 |
|---|---|
| 🟢 온라인 동기화 | 정상 연결됨, 실시간 동기화 중 |
| 🟡 동기화 중 | 데이터 업로드/다운로드 중 |
| 🔴 동기화 오류 | 연결 끊김, 자동 재연결 시도 중 |
| ⬡ 오프라인 | Firebase 미연결 (로컬 전용) |

### 실시간 동기화

- 다른 기기에서 데이터가 변경되면 **새로고침 없이** 자동 반영됩니다
- 변경 감지 시 `🔄 다른 기기에서 변경된 내용이 반영됐습니다` 알림 표시
- 연결이 일시적으로 끊겨도 재연결 후 자동으로 최신 상태로 복구

### 잠금 설정

- 🔒 잠금 버튼: 워크스페이스 ID가 변경되지 않도록 고정
- 잠금 상태에서는 앱 시작 시 자동으로 연결됩니다

### 사용하지 않는 워크스페이스 삭제

1. 🔗 버튼 → 하단 **사용하지 않는 워크스페이스 삭제** 섹션
2. 삭제할 워크스페이스 ID 입력
3. `삭제` 버튼 → 이중 확인 후 Firebase 데이터 영구 삭제

## 설정 및 데이터 관리

**더보기 → 설정** 탭

### 저장공간 관리

- **사용량 바**: 현재 localStorage 사용량 시각화 (한도 ~5MB)
- **재고 이력 즉시 정리**: 어제·오늘 이력만 남기고 삭제
- **오래된 전표 자동 정리**: 완납된 n개월 이상 전표 일괄 삭제

> Firebase 연결 시: 전체 데이터는 서버에 안전하게 보관, 로컬엔 최근 3개월만 유지

### 데이터 관리

| 기능 | 설명 |
|---|---|
| Excel 내보내기 | 전체 납품 내역 .xlsx 저장 |
| JSON 백업 내보내기 | 전체 데이터 .json 저장 |
| JSON 가져오기 | 백업 파일로 복원 |
| 샘플 데이터 생성 | 테스트용 데이터 추가 |
| 전체 데이터 초기화 | ⚠️ 모든 데이터 영구 삭제 |

## PWA 앱 설치

홈 화면에 아이콘을 추가해 앱처럼 사용할 수 있습니다.

### Android (Chrome)

1. Chrome 주소창 우측 `⋮` 메뉴 → **홈 화면에 추가**
2. 또는 주소창 우측 설치 아이콘(📥) 탭

### iOS (Safari)

1. Safari 하단 **공유** 버튼 → **홈 화면에 추가**
2. 이름 확인 후 `추가` 탭

### 설치 후 장점

- 전체화면으로 실행 (주소창·탭바 없음)
- 오프라인에서도 완전히 동작
- 홈 화면에서 앱처럼 바로 실행

## 자주 묻는 질문

**Q. 데이터가 사라졌어요**  
A. Firebase에 연결되어 있다면 🔗 버튼 → 다시 연결하면 서버에서 복원됩니다. Firebase 미연결 상태에서 브라우저 데이터를 삭제했다면 마지막 JSON 백업 파일로 복원하세요.

**Q. 두 기기에서 동시에 입력하면 어떻게 되나요?**  
A. 마지막으로 저장한 기기의 데이터가 우선 적용됩니다. 충돌 방지를 위해 동시 입력은 피해주세요.

**Q. 오프라인에서 입력한 내용이 온라인 복귀 후 사라져요**  
A. 설정 탭 → 워크스페이스 잠금(🔒)을 활성화하면 온라인 복귀 시 오프라인 변경사항을 자동으로 서버에 업로드합니다.

**Q. 워크스페이스 ID를 잊어버렸어요**  
A. 설정 → 앱 정보에서 현재 연결된 워크스페이스 ID를 확인할 수 있습니다.

**Q. 앱이 느려졌어요**  
A. 설정 → 저장공간 관리 → 재고 이력 즉시 정리 또는 오래된 전표 정리를 실행해 보세요.
