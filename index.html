변경 이력
v56 PWA 앱 설치 지원 — 단일 HTML 파일 안에 manifest·아이콘·Service Worker 완전 내장. 크롬에서 열면 "홈화면에 추가" 배너 자동 표시 → 설치 후 주소창 없는 전체화면 앱으로 실행. 오프라인 캐시로 인터넷 없이도 앱 유지. iOS 사파리에서는 공유→홈화면에 추가로 설치 가능.
v55 할인 완납 수금 누계 버그 수정 — 할인 완납 시 실수령액 대신 청구금액으로 수금 통계가 집계되던 문제 수정. _actualPaid() 헬퍼 도입 → 내역·정산·대시보드·미수탭·거래명세표 등 수금액 집계 14곳 일괄 수정. confirmPartialPayDiscount의 paidAmount 저장값도 실수령액으로 수정.
v54 할인 완납 기능 추가 — ① ⚠미수 배지 탭 시 퀵결제 팝업에 "✂️ 끝자리 할인 완납" 영역 추가: 실수령액 입력 → 차액 할인 표시 → 현금/이체 선택 → 완납 처리 ② 입금 처리 모달(거래명세표)에도 동일 기능 추가: 여러 전표에 걸쳐 할인 분배 ③ 전표 카드·거래명세표에 "✂️ 할인완납" 배지 및 할인 금액 표시 ④ 미수 복귀 시 discount 필드 자동 초기화
v53 검색창 더블탭 전체선택 — 납품 거래처·거래처·내역·정산 검색창에서 두 번 터치(모바일) 또는 더블클릭(PC) 시 텍스트 전체 선택. 기존 검색어를 한 번에 지우고 새 거래처를 바로 입력 가능.
v52 코드 품질 개선 4건 — ① changelog v48·v49 HTML 태그 오류 수정(v48 항목 누락·고아 태그 제거) ② document click 리스너 중복 등록 제거(모달 닫기+드롭다운 닫기 핸들러 통합) ③ Firebase 데이터 정규화 헬퍼 함수 추출(_normClientFromFb·_normOrderFromFb) — 4곳 중복 인라인 코드 제거 ④ Firebase 실시간 리스너 공용 핸들러(_fbValueHandler) 추출 — 백업 복원·_doConnect 재사용
v51 정밀 코드 점검 후 추가 수정 6건 — ① 재고 자동차감 OFF여도 품목 신규 등록·수정 가능하도록 차단 해제 ② 캐시 변수 선언 위치 통일(코드 일관성) ③ 수금취소(paidAmount=0) 저장 누락 수정 ④ 대시보드 미수금 이상 데이터 방어 코드 추가 ⑤ 전체완납 수금방법 선택 UX 개선(취소 오인 방지) ⑥ 거래처 미연결 전표 품목 추천 지원
v50 안정성 전면 개선 — ① 저장공간 부족 시 메모리 데이터 보존 ② 재고 이력 30일치 보존 ③ 재고 품목 삭제 항상 허용 ④ 단가 조회 캐시화 ⑤ 수금액 음수 방어 ⑥ 데이터 해시 키 순서 안정화 ⑦ 퀵결제 팝업 ESC 키 지원 ⑧ JSON 백업 버전 표기 수정
v49 납품수정보정 출고 상쇄 버그 수정 — 잘못 입력한 납품을 수정할 때 수정보정(edit_adj +)이 입고로 집계되던 버그 수정. 이제 출고에서 상쇄되어 실입고·실출고 수치가 정확하게 표시됨. (재고 카드, 과거조회, 재고 이력 헤더 3곳 모두 수정)
v48 모달 뒤로가기(이전) 버튼 지원 — 거래 명세서·입금처리 등 모든 팝업을 스마트폰 이전(←) 버튼으로 닫을 수 있게 개선. 팝업이 여러 개 열려 있으면 이전 버튼을 누를 때마다 하나씩 순서대로 닫힘.
v47 재고 이력 시간 KST 보정 — 재고 이력 타임스탬프를 KST 기준으로 올바르게 표시.
v46 납품 확정 후 내역 탭 자동 전환 — 🚚 납품 확정 시 내역 탭(오늘)으로 바로 이동.
v45 내역 탭 수금방법 퀵선택 — 미수 배지 탭 시 💵현금/🏦계좌이체 선택 팝업 표시 후 완납. 완납 배지 탭 시 confirm 후 미수 복귀. 내역 수금액 카드에 현금/이체 금액 분리 표시. 전체완납 시에도 수금방법 선택.
v44 수금 방법 구분 추가 — 입금처리·수금수정 모달에 💵현금/🏦계좌이체/📝기타 버튼 선택. 거래명세표 수금 이력에 방법 배지 표시. Excel 내보내기에 수금방법 컬럼 추가.
v43 미수금 관리 강화 — 🔴미수 전용 탭 추가(거래처별 미수금·경과일·전화/문자/입금처리 원스톱), 거래처 카드 미수금 색띠+배지(30/60/90일 기준), 대시보드 미수 거래처 TOP 현황, 네비게이션 미수 탭 배지.
v42 전체 코드 점검 및 버그 수정 4건 — ① _trimLogByDate 타임존 버그 수정(kstAddDays 사용) ② _compactTs undefined 입력 시 빈 문자열 대신 null 반환·_minifyOrder에서 조건부 저장 ③ normalizeBackupData 백업 복원 시 items.total 미복원 수정 ④ _minifyOrder createdAt 항상 포함되던 문제 수정.
v41 localStorage 4종 최적화 — ① 전표 품목 total 필드 제거(로드 시 qty×price 자동복원) ② createdAt/updatedAt 24→16자 단축 ③ 빈 note/paidNote 저장 방지 + 비연결 모드도 항상 _minifyOrder 적용 ④ prices 최근 6개월 미사용 단가 자동 정리. 평균 추가 15~25% 절감.
v40 재고 이력 보관 정책 변경 — 어제·오늘 이력만 유지하고 그 이전 이력 자동 삭제. 모든 이력 추가 지점(납품차감·수동입출고·편집보정·이월) 및 긴급 정리·경량 저장 모두 동일 정책 적용.
v39 localStorage 저장 최적화 — _minifyOrder/_minifyClient로 중복 필드 제거 · 완납+1개월 초과 전표 items 로컬 저장 제외 · 재고 이력 10→5개 축소. 평균 20~40% 절감. 거래명세표 카카오톡 공유 버튼 추가 (Web Share API, 클립보드 폴백). 버그 수정: _noItems Firebase 오염 방지 · Excel 내보내기 누락 · 거래명세서 품목 빈칸.
